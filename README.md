# Bootkit Analysis Framework

## 개요
이 프레임워크는 부트킷 악성코드의 동작 방식을 이해하고 분석하기 위한 도구로, 실제 악성코드가 아닌 보안 연구 및 방어 메커니즘 개발을 위한 교육용 프레임워크입니다.


## 요구사항

### 사용자 모드 컴포넌트
- Windows 10/11
- Visual Studio 2019 이상
- C++ 데스크톱 개발 워크로드

### 드라이버 컴포넌트
- 위의 모든 요구사항 +
- Windows Driver Kit (WDK) 10
- Windows SDK 10
- 테스트 서명 모드 활성화

## 프로젝트 구조
```
bootkit-analysis-framework/
├── drivers/              # 커널 드라이버 프로젝트
│   └── BootkitDriver/    # 메인 분석 드라이버
├── tools/                # 개발 도구 및 유틸리티
│   └── BootkitTools/     # 분석 도구
├── tests/                # 테스트 프로젝트
│   └── BootkitTests/     # 테스트 스위트
```
## 주요 기능

### 1. 커널 레벨 분석 기능

#### IRP 디스패처 모니터링
- I/O Request Packet (IRP) 통신 흐름 실시간 추적
- 드라이버 간 통신 패턴 분석
- IRP 디스패치 및 완료 루틴 모니터링
- 비정상적인 IRP 처리 탐지

#### 커널 후킹 프레임워크
- SSDT (System Service Descriptor Table) 후킹 탐지
- IDT (Interrupt Descriptor Table) 변조 감지
- 인라인 후킹 및 트램폴린 탐지
- 커널 함수 포인터 무결성 검증

#### 프로세스 은폐 탐지
- EPROCESS 구조체 직접 스캔
- 숨겨진 프로세스 발견 (DKOM 기법)
- 프로세스 링크 리스트 무결성 검사
- 루트킷 프로세스 은폐 기법 분석

### 2. 부트 프로세스 분석

#### 부트 플로우 인터셉터
- 부팅 단계별 실행 흐름 추적
- 부트 섹터 및 MBR 분석
- 부트로더 실행 과정 모니터링
- 부트킷 감염 지점 식별

#### UEFI 분석 엔진
- UEFI 펌웨어 구조 분석
- UEFI 부트 서비스 및 런타임 서비스 모니터링
- UEFI 변수 무결성 검증
- Secure Boot 우회 기법 탐지

#### 부트로더 엔트리 포인트 분석
- 부트로더 진입점 식별
- 부트 체인 무결성 검증
- 부트 과정 중 코드 실행 추적

### 3. 회피 기법 탐지

#### 안티-디버깅 탐지
- 디버거 존재 여부 확인 기법 탐지
- 타이밍 기반 디버깅 탐지 우회
- 하드웨어 브레이크포인트 탐지

#### 안티-VM 탐지
- 가상 머신 환경 감지 기법 분석
- VMware, VirtualBox, Hyper-V 탐지 우회
- 하드웨어 시그니처 기반 VM 탐지

#### 안티-포렌식 기법
- 메모리 덤프 방지 기법 분석
- 로그 삭제 및 은폐 탐지
- 타임스탬프 조작 감지

### 4. 고급 분석 기능

#### Discord 웹훅 매니저
- 분석 결과 실시간 알림
- 이벤트 기반 리포팅
- 원격 모니터링 지원

#### DPAPI 분석기
- Windows Data Protection API 분석
- 암호화된 자격 증명 추출 시뮬레이션
- 마스터 키 복구 프로세스 분석

#### 자율 의사결정 엔진
- 행위 기반 위협 분석
- 자동화된 대응 시나리오
- 머신러닝 기반 패턴 인식

### 5. 개발 도구

#### BootkitTools
분석 및 모니터링을 위한 명령줄 도구:
- `analyze` - 부트킷 동작 패턴 분석
- `monitor` - 실시간 시스템 모니터링
- `report` - 상세 분석 리포트 생성
- 커스터마이징 가능한 분석 규칙

#### BootkitTests
자동화된 테스트 스위트:
- 회피 엔진 테스트
- SSDT 후킹 테스트
- IRP 모니터 테스트
- 통합 테스트 프레임워크

### 6. 통신 및 인터페이스

#### 통신 어댑터
- 커널-유저 모드 통신
- IOCTL 기반 제어 인터페이스
- 이벤트 기반 알림 시스템

#### 동시성 관리
- 스레드 안전 데이터 구조
- 스핀락 및 뮤텍스 관리
- 비동기 작업 처리

## 디버깅
- WinDbg Preview 사용 권장
- 네트워크 또는 시리얼 커널 디버깅 설정
- 드라이버 진입점에 중단점 설정
- IRP 디스패치 및 완료 모니터링
